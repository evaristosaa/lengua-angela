# Repasa Lengua con Angela Constitution

## Core Principles

### I. Child-first learning

Every feature must serve a 9-year-old learner preparing 4th grade Spanish Language. The app must feel playful, encouraging and clear. Educational value beats novelty.

### II. Static-first delivery

The app must be deliverable as a static web app suitable for GitHub Pages. No mandatory backend, paid service or private server may be required for the MVP.

### III. Parent-visible progress

Learning progress must be understandable by adults without turning them into administrators. Parent views should show time, completed missions, strengths, weak areas and suggested next review.

### IV. Safe, private by default

Progress, recordings and child-created texts must stay local in the MVP unless an explicit export/import action is chosen. No external sharing is allowed by default.

### V. Incremental, demonstrable slices

Each user story must be independently demonstrable. The first slice must deliver a usable daily mission with a visible unicorn reward, not only scaffolding.

## Product Constraints

- Content must be adapted to 4th grade, starting with a 3rd-to-4th bridge.
- The unicorn reward system and comic Angela character are core experience elements, not decoration.
- The MVP must include enough local content to be usable offline after load.
- Visual assets from Angela will be integrated later as explicit assets; placeholders are allowed until they arrive.

## Development Workflow

- Generate specification, plan and tasks before implementation.
- Implementation will be done later with Codex CLI and Superpowers.
- Validate any implementation with local build and browser checks before publishing.

## Governance

This constitution guides specification and planning for this project. Changes that affect scope, child privacy, GitHub Pages deployability or MVP priority must be reflected in spec, plan and tasks.
